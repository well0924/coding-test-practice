이 문제는 펠린드롬을 이해를 했는지를 묻는 문제입니다.  펠린드롬은 **앞에서 읽은 문자열과 뒤에서 읽은 문자열이 동일한 경우**입니다. 즉, 문자열의 **양 끝 문자를 순차적으로 비교**하면 된다.

이 문제에서는 char[] 로 했습니다. 
- 문자열은 인덱스 접근이 불편하므로`toCharArray()`를 사용해 **char 배열로 변환**했습니다. 
- 이렇게 변환한 다음에 앞(`i`)과 뒤(`length - 1 - i`)를 쉽게 비교할 수 있어서 char[]로 했습니다.

위의 앞과 뒤를 비교하는 문법은 아래와 같습니다. 

```
if(arr[i] == arr[arr.length-1-i])
```

그리고 변환된 배열을 전부 다 비교를 할 필요는 없고 문자열 길이의 절반만 비교를 했습니다. 이유는 배열의 앞쪽 절반과 뒤쪽 절반이 서로 대응이 되기 때문입니다.

코드를 작성하는 순서는 다음과 같습니다.

- Scanner를 사용해서 문자열을 받는다.
- 문자열을 toCharArray를 사용해서 char[]로 전환을 한다.
- 반복문을 사용해서 배열의  절반만 돌려서 펠린드롬 조건으로 맞지 않으면 break로 하고 결과값을 0으로 한다.
- 그 외의 값은 1로 한다.
- 최종적인 result 값을 출력한다.

아래는 작성 순서를 플로우  차트로 그린 것이다.

<img width="1388" height="710" alt="Image" src="https://github.com/user-attachments/assets/3029ed54-311b-4d20-beda-fbea04020672" />

최종적인 코드는 다음과  같습니다.
```
	public static void main (String[]args) {
		Scanner sc = new Scanner(System.in);
		String input = sc.next();
		// 입력 받은 값을 char타입의 배열로 바꾸기.
		char[] arr = input.toCharArray();
		int result = 1;
		
		for(int i=0; i<arr.length/2;i++) {
			//조건에 맞지 않는 경우에는 break 
			if(arr[i]!=arr[arr.length-1-i]){
				result = 0;
				break;
			}
		}
		System.out.println(result);
	}
```

3.소감

이번 문제를 통해 펠린드롬의 개념 자체는 이해하고 있었지만,  이를 **간결한 코드와 명확한 조건식으로 표현하는 연습이 더 필요하다는 점**을 느꼈다.

문제를 처음 접근할 때는 전체 흐름을 먼저 생각했고, 그 과정에서 문자열을 배열로 변환하고 양 끝을 비교하는 방식으로 해결할 수 있었다.  

다만 코드를 작성하면서 **불필요한 변수 처리나 반복적인 대입을 줄이면 더 깔끔한 코드가 될 수 있겠다는 점**을 확인했다.

앞으로는 문제를 해결하는 것에 그치지 않고, **코테 환경에서 읽기 쉽고 의도가 바로 드러나는 코드로 정리하는 연습**을 더 해볼 예정이다.
